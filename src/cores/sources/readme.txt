# sources

该目录实现 **流量源（Source）**，负责在仿真中生成并注入数据包/Flit 到网络中，是 NI（Network Interface）里发送端的核心组成部分。

## 目录职责

`sources` 目录中的模块主要负责：

- 根据配置的业务模型生成报文
- 控制报文注入速率、长度、目的地址等属性
- 将生成的流量送入 `NI`，再由 `NI` 注入 NoC
- 统计发送侧相关行为

## 主要文件

### `Source_Ifc.ned`
定义 Source 模块的统一接口。  
其他具体 Source 实现都应遵循这个接口，以便在 `NI.ned` 中通过参数化方式动态装配。

### `PktFifoSrc.ned`
`PktFifoSrc` 的 NED 结构定义文件。  
用于描述该流量源模块的参数、门（gates）以及与 OMNeT++ 仿真框架的集成方式。

### `PktFifoSrc.h`
`PktFifoSrc` 的类声明文件。  
定义该模块的成员变量、状态、统计量及主要成员函数。

### `PktFifoSrc.cc`
`PktFifoSrc` 的具体实现文件。  
负责实际的数据包生成、缓存、注入控制以及与下游模块的交互逻辑。

## 模块定位

在整体结构中，`sources` 目录下的模块通常作为：

- **处理核/任务端的发送器**
- **NI 的发送子模块**
- **网络业务注入入口**

典型连接关系如下：

`Core/Task -> Source -> NI -> Router/Network`

## 设计特点

- 采用接口 + 实现分离方式，便于替换不同流量模型
- 可扩展为不同业务模式，例如：
  - 固定速率注入
  - 泊松流量
  - 突发流量
  - 任务驱动流量
- 适合作为后续 ONoC / 可重构 NoC 中的数据注入入口

## 备注

当前目录中的核心实现是 `PktFifoSrc`，从命名看它是一个基于 FIFO/分组队列思想的流量源模块，适合用于分组网络仿真中的发送端建模。
